# Behavior baseline

This slice freezes observable Codex Termux wrapper behavior before role-oriented layout changes.

## Provenance

The initial fixture set was reviewed and selectively rebuilt from archived PR #5 at `archive/pr-5-golden-runtime-contracts-20260717` (`4157d3baff8548e2586d95efd367f02589339802`). The archived branch is evidence, not a merge dependency.

## Fixture classes

- `public`: CLI routing, profile persistence, and public error surfaces.
- `internal-contract`: state/registry serialization and runtime process contracts that downstream refactors must preserve.
- `white-box-transition`: transaction rollback behavior. These fixtures may call private transition helpers deliberately and must be updated when internal transaction boundaries are intentionally redesigned.

## Safety properties

The capture harness uses an explicit environment allowlist. It does not inherit credentials, tokens, SSH agent variables, or unrelated host configuration into fixture subprocesses.

Expected outputs are normalized for repository, sandbox, home, prefix, and temporary paths. Golden updates must be generated by an intentional `--update` run and reviewed as code changes.
